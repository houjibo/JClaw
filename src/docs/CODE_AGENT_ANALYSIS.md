## ❌ JClaw 缺失的核心 Code Agent 能力

### 1. 文件编辑（严重缺失） 🔴

```typescript
file_edit path="src/main.java" diff="""
- old code
+ new code
"""
```

**JClaw 现状**: 只能全量覆盖写入
```java
// JClaw 仅支持
file_write path="src/main.java" content="完整内容"
```

**影响**: 无法进行精确的代码修改，容易破坏原有代码结构

**优先级**: 🔴 **最高** - 必须实现

---

### 2. 多轮对话上下文（严重缺失） 🔴

- 完整的对话历史管理
- 上下文窗口管理
- 引用之前的文件和修改

**JClaw 现状**: 无状态，每次请求独立

**影响**: 无法进行连续的代码开发任务

**优先级**: 🔴 **最高** - 必须实现

---

### 3. 代码审查（重要缺失） 🟡

- 代码质量分析
- 潜在问题检测
- 改进建议

**JClaw 现状**: 无

**优先级**: 🟡 **中等** - 重要但可延后

---

### 4. 网页抓取（次要缺失） 🟢

- 抓取 URL 内容
- 提取文本

**JClaw 现状**: 无

**优先级**: 🟢 **低** - 可选功能

### 5. 成本追踪（次要缺失） 🟢

- API 调用成本实时追踪

## 📋 升级为成熟 Code Agent 的路线图

### Phase 1 - 核心能力补全（1-2 周）🔴

1. **实现 FileEditTool**
   - 支持 diff 格式
   - 支持多处同时修改
   - 支持创建/删除函数/类
   
2. **实现上下文管理**
   - 会话状态存储
   - 文件修改历史
   - 多轮对话支持

### Phase 2 - 高级能力（2-4 周）🟡

3. **实现代码审查**
   - 代码坏味道检测
   - 最佳实践建议
   - 安全漏洞扫描

4. **实现 WebFetchTool**
   - URL 内容抓取
   - Markdown 提取

### Phase 3 - 增强能力（1-2 月）🟢

5. **LSP 集成**
   - 代码跳转
   - 符号查找
   - 错误诊断

6. **成本追踪**
   - Token 统计
   - API 成本计算

## 🎯 结论

### JClaw 当前定位

**✅ 合格的工具执行器** - 9 个工具覆盖核心场景
**🟡 初级 Code Agent** - 缺少关键的文件编辑和上下文管理
**❌ 非完整 Code Agent** - 无法独立承担复杂开发任务

### 核心缺失

1. **FileEditTool** - 无法精确修改代码
2. **上下文管理** - 无法进行多轮对话开发

### 建议

**短期（1-2 周）**:
- 优先实现 `FileEditTool`（diff 编辑）
- 实现基础会话上下文管理

**中期（1 月）**:
- 实现代码审查功能
- 添加 LSP 支持

**长期（2-3 月）**:
- 开发 CLI 终端界面
- 实现多 Agent 协调
